¿Qué es Devlmer?

Un instalador inteligente que convierte cualquier proyecto en un entorno profesional para Claude Code

Ecosistema Profesional Automático

Devlmer analiza tu proyecto, detecta el stack tecnológico y el dominio empresarial, y configura automáticamente todo lo que necesitas: skills con mejores prácticas enterprise, MCP servers para integrar servicios externos, agents especializados, hooks de verificación y slash commands — todo en un solo paso.

  • Detecta tu stack: lenguajes, frameworks, bases de datos, servicios
  • Instala skills profesionales adaptadas a tu dominio
  • Configura MCP servers para GitHub, Slack, Stripe, Sentry y más
  • Crea agents especializados según tu tipo de proyecto
  • Genera slash commands (/comando) para cada skill
  • Incluye CLI global (dee) para gestionar desde la terminal
$ dee install ~/Projects/mi-app

Cómo Funciona

Pipeline automático de detección, instalación y activación

1

Fingerprinting

El script detect_project.py analiza tu proyecto: lee package.json, requirements.txt, Dockerfile, estructura de carpetas y más. Identifica 157+ tecnologías y 22 dominios empresariales.

2

Blueprint Matching

El orchestrator cruza tu perfil con ecosystems.json — una librería de blueprints optimizados por dominio. Selecciona los skills, MCPs y agents óptimos para tu caso.

3

Instalación

Instala todo en .claude/: skills con contenido enterprise, MCP servers pre-configurados, agents con instrucciones detalladas, hooks de SessionStart y PostToolUse.

4

Slash Commands

Cada skill se registra automáticamente como un /comando en Claude Code. Además, se instalan 3 comandos DEE: /dee-demo, /dee-status, /dee-doctor.

5

Setup Wizard

Un asistente interactivo te guía para configurar API keys de cada servicio: GitHub, Slack, Stripe, Sentry, etc. Explica qué hace cada uno y dónde encontrar tu key.

6

Activación en Claude Code

Al abrir tu proyecto en Claude Code, el hook SessionStart muestra un dashboard con todo lo instalado. Escribe / para ver todos los comandos disponibles.

Instalación

Dos pasos: instala el CLI y luego configura tu proyecto

Paso 1 — Instala el CLI

Un comando instala dee globalmente en tu sistema. Solo se hace una vez.

curl -fsSL https://raw.githubusercontent.com/Soyelijah/devlmer-ecosystem-engine/main/install-cli.sh | bash

Compatible con macOS (Intel y Apple Silicon) y Linux. Detecta tu sistema automáticamente.

Paso 2 — Configura tu proyecto

Usa dee install en cualquier proyecto para generar su ecosistema

# Instalar en un proyecto específico: dee install ~/Projects/mi-app # O en la carpeta actual: dee install

Detecta tu stack, instala 62 skills, 23+ MCPs, agents, hooks y slash commands en menos de 30 segundos.

Paso 3 — Configura API Keys (opcional)

El setup wizard te guía servicio por servicio

# Aparece automáticamente al final de la instalación # Cada servicio tiene explicación + link directo a la API key # Puedes saltarlo y configurar después con: bash ".claude/setup-wizard.sh" ~/Projects/mi-app
Paso 4 — Abre Claude Code y explora

Tu ecosistema se activa automáticamente al abrir el proyecto

cd ~/Projects/mi-app && claude # Al abrir se muestra el dashboard automático: # 🧠 DEVLMER ECOSYSTEM ENGINE v3.0 # 📊 Project: mi-app | Domain: SaaS # ⚡ 22 skills | 24 commands | 14 agents | 23 MCPs # # Comandos para empezar: # /dee-demo → Tour interactivo del ecosistema # /dee-status → Dashboard de todo lo instalado # /dee-doctor → Health check con score 0-100 # # Desde la terminal: # dee help → Ver todos los comandos disponibles # dee doctor → Diagnóstico completo # dee update → Actualizar ecosistema del proyecto # dee self-update → Actualizar el CLI a la última versión

También puedes instalar sin el CLI: rm -rf /tmp/dee && git clone https://github.com/Soyelijah/devlmer-ecosystem-engine.git /tmp/dee && bash /tmp/dee/install.sh ~/Projects/mi-app

Actualizar

Mantén tu ecosistema al día sin perder configuración

CLI dee — Recomendado

El instalador agrega el comando dee globalmente. Un solo comando para todo:

# Actualizar el ecosistema del proyecto actual: dee update # Actualizar un proyecto específico: dee update ~/Projects/mi-app # Actualizar el CLI dee a la última versión: dee self-update # Otros comandos del CLI: dee status # Ver componentes instalados dee doctor # Health check y diagnóstico dee install ~/nuevo-app # Instalar en proyecto nuevo dee uninstall # Desinstalar del proyecto actual dee version # Ver versión instalada dee help # Ver todos los comandos disponibles
One-liner alternativo

Si no tienes el CLI (instalaciones anteriores a v3.0):

rm -rf /tmp/dee && git clone https://github.com/Soyelijah/devlmer-ecosystem-engine.git /tmp/dee && bash /tmp/dee/update.sh ~/Projects/mi-app
