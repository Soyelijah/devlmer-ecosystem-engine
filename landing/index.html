¿Qué es Devlmer?

La solución inteligente para ecosistemas profesionales automáticos

Automatización Completa del Ecosistema

Devlmer detecta automáticamente el tipo de proyecto en el que estás trabajando y genera un ecosistema profesional completo en minutos. Olvídate de configuraciones manuales y disfruta de skills, MCPs y agents configurados con mejores prácticas enterprise.

  • Detecta automáticamente tu stack tecnológico
  • Instala skills profesionales específicas
  • Instala y configura MCPs específicos para tu stack
  • Crea agents especializados para tu dominio
  • Genera blueprints de mejores prácticas
  • Se conecta con tu cuenta de GitHub

Cómo Funciona

Seis pasos automáticos para un ecosistema profesional completo

Paso 1

Ejecuta el Instalador

Un único comando bash install.sh que inicia todo el proceso automático de detección e instalación.

Paso 2

Detección Automática

El fingerprinter analiza tu proyecto y detecta tecnologías, frameworks, dominio empresarial y nivel de madurez.

Paso 3

Instalación Completa

Se instalan 21+ skills, se configuran MCPs personalizados, se crean agents especializados y slash commands.

Paso 4

Setup Wizard

Un asistente guiado paso a paso te ayuda a configurar tus API keys con explicaciones claras de cada servicio.

Paso 5

Slash Commands Listos

Cada skill se registra como un /comando en Claude Code. Escribe / y elige tu skill al instante.

Paso 6

¡Todo Visible en Claude Code!

Tus skills aparecen en /skills, MCPs en /mcp, y plugins en /plugin. Todo integrado nativamente.

Cómo Instalar

Flujo completo: de cero a ecosistema profesional en minutos

Paso 1 — Ejecuta un solo comando

Copia este comando en tu terminal. Reemplaza la ruta con la de tu proyecto.

# Ejemplo con ruta real: rm -rf /tmp/dee && git clone https://github.com/Soyelijah/devlmer-ecosystem-engine.git /tmp/dee && bash /tmp/dee/install.sh ~/Projects/mi-app # También funciona con la carpeta actual: rm -rf /tmp/dee && git clone https://github.com/Soyelijah/devlmer-ecosystem-engine.git /tmp/dee && bash /tmp/dee/install.sh .
Paso 2 — Espera la instalación automática

El engine detecta tu stack, instala skills, MCPs, agents y slash commands

# Todo es automático — tarda menos de 30 segundos # Detecta: lenguajes, frameworks, arquitectura, dominio del proyecto # Instala: skills, slash commands, agents, MCPs, hooks # Genera: CLAUDE.md, settings.json, PROJECT_PROFILE.json
Paso 3 — Configura API Keys (opcional)

Al final aparece un asistente que te guía para configurar tus servicios

# El wizard aparece al final de la instalación # Te explica cada servicio y dónde encontrar la API key # Puedes saltarlo y configurar después con: bash ".claude/setup-wizard.sh" ~/Projects/mi-app
Paso 4 — Abre Claude Code

El ecosistema se activa automáticamente al abrir tu proyecto

cd ~/Projects/mi-app && claude # 🧠 DEVLMER ECOSYSTEM ENGINE v3.0 # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ # 📊 Project: mi-app | Domain: Saas # ⚡ 22 skills | 24 commands | 14 agents | 22 MCPs # # 💡 /dee-demo (tour) | /dee-status (dashboard) | /dee-doctor (salud) # Escribe / para ver todos los comandos disponibles
Paso 5 — Explora y actualiza

3 comandos dentro de Claude Code + el CLI dee para gestionar desde la terminal

# Dentro de Claude Code: # /dee-demo → Tour interactivo (¡empieza aquí!) # /dee-status → Dashboard de todo lo instalado # /dee-doctor → Health check con score 0-100 # # Desde la terminal (CLI instalado automáticamente): # dee update → Actualizar skills y comandos # dee status → Ver qué hay instalado # dee doctor → Diagnóstico y salud del ecosistema

Actualizar Instalación Existente

Un comando corto — como brew o npm

Opción 1 — CLI dee (recomendado)

El instalador agrega el comando dee globalmente. Solo escribe:

dee update

Otros comandos disponibles:

dee status # Ver qué hay instalado dee doctor # Health check y diagnóstico dee update ~/otro-app # Actualizar otro proyecto dee install ~/nuevo-app # Instalar en proyecto nuevo dee uninstall # Desinstalar del proyecto actual dee version # Ver versión instalada
Opción 2 — One-liner (si no tienes el CLI)

Si por alguna razón no se instaló el CLI, usa el one-liner:

rm -rf /tmp/dee && git clone https://github.com/Soyelijah/devlmer-ecosystem-engine.git /tmp/dee && bash /tmp/dee/update.sh "/path/to/your/project"
